# finoject 法令ビューア（資金決済・犯収法・金商法）

暗号資産・決済の中核となる法令・関連政省令を **e-Gov 法令API v2** から取得し、
条文を検索・参照しやすく表示する静的サイト。毎朝07:35（JST）に自動更新。

## 対象法令（12）
資金決済法／同施行令／暗号資産交換業者府令／資金移動業者府令／前払式支払手段府令／電子決済手段等取引業者府令／
犯収法／同施行令／同施行規則／金融商品取引法／同施行令／金融商品取引業等府令

## 仕組み
- `law-viewer/fetcher.js` … e-Gov API v2 (`/law_data/{law_id}`) から本文を取得し、条文(Article)・章節見出しに構造化して `law-viewer-site/data/{law_id}.json` を生成。`law_revision_id` を前回と比較し更新法令を検出。
- `law-viewer-site/index.html` … 法令一覧＋条文表示＋条文内キーワード検索。各条はe-Gov原文へリンク。`?law=ID&art=第九条` でディープリンク可。
- `.github/workflows/crawl.yml` … 毎朝07:35 JSTにcron実行 → 取得 → data更新コミット → GitHub Pages公開。

## 補足
e-Govは「法令（法律・政令・府省令）」のみ。金融庁の監督指針・事務ガイドライン・パブコメ・新旧対照表は法令ではないため別途（規制ウォッチ側／専用レイヤーで）取得する。

## 手動実行
Actionsタブ →「fetch-laws-and-publish」→「Run workflow」。ローカルは `cd law-viewer && node fetcher.js`。

## 表示ルール（恒久仕様・毎更新で維持／追加法令にも自動適用）
これらは `fetcher.js`／`index.html` の汎用ロジックで実装されており、毎朝の更新でも、将来法令・ガイドラインを追加しても自動で適用される。**変更・劣化させないこと。**
1. 漢数字→算用数字（第三十三条→第33条、二百八十七→287）。数詞・桁・枝番（第63条の二→第63条の2、連鎖枝番も）・複合単位（三事業年度→3事業年度）に対応。「一般」「十分」「第三者」「四半期」等の語は保護。
2. 分数・歩合（百分の二→100分の2、二分の一→2分の1）。
3. 5桁以上の数値に3桁カンマ（10000000→10,000,000）。
4. 全角英字→半角（ＦＡＴＦ→FATF）、全角ハイフン→半角（Ⅱ－２→Ⅱ-2）。
5. 条文の項/号/イロハを改行＋インデント。定義号はColumn（用語｜定義）間に半角スペース（番号等 番号）。
6. ガイドライン(PDF)は箇条書き/ラベル（①②、**⑴⑵⑶…⑽**、イ．、・○●、（注）（参考）（別紙等）の行頭で改行。`⑴`等（括弧付き数字 U+2474〜）はpdftotextが1行に連結するため、行頭以外でも各マーカーの前で改行する。ラベルのみの行（（注4）等）は次行を連結。さらに**本文に埋もれたローマ数字の章見出し（例「Ⅱ全ての種別の資金移動業者に共通する監督上の評価項目」＝ローマ数字＋ダッシュ/数字以外で始まる題）は独立した見出しブロックに分離**（`fetcher.js` の `G_CHAP`＋`。Ⅱ…`の前で改行）。節見出し（Ⅰ-2-2-2…）は `G_HEAD`。
7. カッコ（）の入れ子の深さで背景色分け（フロント `paint()`／CSS `.p1..p6`）。検索ハイライトと併用。番号・記号だけのカッコ（（1）（一）（イ）等＝条文番号）は色付けしない（中身のある節カッコのみ着色）。
8. 本文中の条文参照をリンク化（フロント `render()` が1パスで色分け・検索・参照リンクを処理）。**参照リンクはすべて実URL＋`target="_blank"`で新しいタブで開く**（左クリック=新タブで現在地を保持、右クリック=ブラウザ標準の「新しいタブ/ウィンドウで開く」。SPAでの画面内遷移は行わない＝迷子・戻れない問題を回避）。ディープリンクは `?law=ID&art=第N条`（条）／`?law=ID&art=第N条&para=N`（項）。
   - 「法第N条（の2／第M項／第K号）」→ その文書の**親法令**の該当条文。親法令は `index.html` の `LAW_OF`（文書law_id→親法令id）で解決＝府令・省令・ガイドラインごとに「法」が違っても正しく対応。
   - 「第N条…」（法・名称法を伴わない裸の参照）→ **その文書自身**の該当条文。ただし**自文書に実在する条のときだけ**リンク（他法を引用した文中の「第N条」を自条と誤リンクしない）。
   - **「第N項…」（条を含まない項・号レベルの参照。例: 第4項第2号ロ／第6項各号）→ その条文内の該当項へジャンプ**（条文内スクロール＋項番号を強調）。`render()` が行頭の項番号を `<span class="pa" id="{条番号}:p{N}">` で包んで項アンカーを作り、`第N項…` リンクの `data-para` から `getElementById` でスクロールする。第1項は番号が無いため本文先頭に空アンカー。
   - 「法」「第」の直前が漢字・閉じカッコの場合は対象外＝名称法（民法・会社法・○○法第N条）や（…号）第N条 を誤リンクしない。名称法どうしの相互リンクは将来課題（現状は安全側で非リンク）。
   - ⚠️ paint等で**制御文字センチネルは絶対に使わない**（Read/Edit/シェルで欠落しnullバイト混入→構文破壊の事故あり）。

## 新しい法令・ガイドラインの追加手順
上記ルールは汎用処理なので、**設定に1行追加するだけ**で全機能（番号整形・カッコ色分け・検索・ディープリンク等）が自動適用される。
- 法令（例: 会社法・民法）: e-Govで law_id を確認（`/laws?law_title=会社法` 前方一致）し、`fetcher.js` の `TARGETS` に `{ law_id, group, type }` を追加。新しい `group`（例「会社法」）はサイドバー・並び順とも自動で末尾に表示される（`index.html` の `G_ORDER`／`fetcher.js` の `G` に足せば任意の位置に）。
- ガイドライン: `fetcher.js` の `GUIDELINES` にPDFの `{ key, file, title }` を追加。
- 規制ウォッチ→法令ビューアの条文リンクも効かせたい場合は、`finoject-reg-monitor` の `LAWREFS` に法令名→law_id を追加。
- 追加する文書（府令・省令・ガイドライン等）の本文中「法」が特定の親法令を指す場合は、`index.html` の `LAW_OF` に `{ その文書のlaw_id: 親法令のlaw_id }` を追加（本文中「法第N条」リンクが正しい親法を指すようになる）。
